# Security Policy

## Reporting a vulnerability

Do not open a public issue for an unpatched vulnerability. Use the repository's **Security** tab and create a private vulnerability report through GitHub Security Advisories.

Include the affected version, reproduction steps, impact, and any proposed mitigation. Avoid including real credentials, payment identifiers, private keys, or personal data.

## Supported versions

Security fixes are applied to the latest release and the `main` branch.
